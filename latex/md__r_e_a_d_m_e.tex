This project is a library (\doxylink{flag__module_8h}{flag\+\_\+module.\+h} file with the extra layer \doxylink{usr__flags_8h}{usr\+\_\+flags.\+h} file) which helps to generate flags of 1 bit size.

The project purpose is to facilitate the creation of flags with a 1 bit size, because in some cases, such as in the embedded systems, resources are more limited. Therefore, you have to take care of this aspect. If you are going to use flags in your project, this library can help you to save some memory.\hypertarget{md__r_e_a_d_m_e_autotoc_md1}{}\doxysection{\texorpdfstring{Usage example}{Usage example}}\label{md__r_e_a_d_m_e_autotoc_md1}
The first thing that you need to do is map your flags in the extra layer \doxylink{usr__flags_8h}{usr\+\_\+flags.\+h} (you can create your own .h file based on the structure of the \doxylink{usr__flags_8h}{usr\+\_\+flags.\+h} file and add it to the project build)

The following code is an example of what the extra layer file should look like\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\#ifndef\ \_USR\_FLAGS\_}
\DoxyCodeLine{\#define\ \_USR\_FLAGS\_}
\DoxyCodeLine{}
\DoxyCodeLine{\#include\ "{}flag\_module.h"{}}
\DoxyCodeLine{}
\DoxyCodeLine{\#define\ myFlag\ FLAG\_0}
\DoxyCodeLine{\#define\ ursFlag\ FLAG\_1}
\DoxyCodeLine{\#define\ nameFlag\ FLAG\_2}
\DoxyCodeLine{}
\DoxyCodeLine{\#endif\ //\_USR\_FLAGS\_}

\end{DoxyCode}
 In the example above we specify that the original "{}\+FLAG\+\_\+0"{} name is going to be renamed "{}my\+Flag"{}, or on the other words, we linked the "{}\+FLAG\+\_\+0"{} MACRO with my\+Flag MACRO.

After the user flag mapping, you can add the extra layer to your main project.
